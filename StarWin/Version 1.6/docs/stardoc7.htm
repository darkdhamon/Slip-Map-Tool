Civilization Generator is an utility that simulates the space colonization of the different alien races.
To use it choose a Star file, I recommand you to copy the sector files somewhere, in the case you
didn't like the Empire Gen results.

Press the Go button to begin the process, Empire makes one pass for each century before completing its job.

What's new ?

CivGen 1.6 is now remplacing Empire Generator v1.57a as the main alien empire expansion simulator.
New features of CivGen V1.6 :

  • 16 new random events.
  • No more anachronic unusual abilities (like computer skill bonus for TL 1 species).
  • A RP report saved in a csv file.
  • Events could be saved in a csv file.
  • Mechanoid races are considered like extra sector races, so they will appear only in the last centuries.
  • The less evoluted race in a trade or non intercourse relation gains RP.
  • Wars and science bonus skill gives more RP each centuries.
  • Creation of an alien backup file (*.aln1), so when civilizations are cleared, all the old aliens attributes are restored.
  • The big sector bug should be fixed now with a new main procedure in the program, and it should be quicker.

 

What does Civilization Generator ?

First it populates planets with native alien races, second the races with an adequate Tech Level will colonize near planets and systems. These races will eventually meet each other, makes alliance (with some Tech Level benefit sometimes) or even war (with conquest of opponent's colonies).
Some unsatisfied colonies could also choose to be independant.
 

Clear button

You don't like the empire results or you add new system to the sector after creating empire colonies, you could clear all the results, and rerun again Civilization Generator on a sector. Just click on the Clear button to erase any empire informations.

Insert Alien button

Well you want a race that rules an empire with a size of 1000 UA but don't want to create a sector of this size because StarGen don't permit it, or you don't have a GigaHz processor to generate it quickly or you don't have a disk that could handle 50 Gigas of datas, well here's the Insert Alien function. With this you could import an alien race of a sector to an another. Enter the original race sector file, the alien race ID in the orginial sector,the first colonized world ID and the colony age in centuries. Note that the colony age can't be higher than the alien spacial age (how a race that discovered star travel 5 centuries ago could colonized a planet 9 centuries ago ?), and of course only TL 6+ race could be imported.

Ins Relation button

Lets you choose the initial diplomatic relationship between two species.

History logfile

If this option is checked, a <sector filename>.his textfile is created that resumes all that happens in the sector during the empire generating process.

Quick Method

If this option is checked, the empire generating process will be quicker but less accurate (for example subjugated colonies couldn't be retaken).

Random events

If this option is checked random events could happen during the empire generating process (mainly catastrophes for the generated colonies)

Liberation of allied worlds

If this option is checked, subjugated colonies could be liberated by allied races (this slows the program).

Parameters

Victory Value is the difference of military power needed in order to conquer a planet. For example if Victory Value is set to 2, the attacker need to a have a military power two times superior to the defender to win the battle.
Ranges represent the average expansion speed in AU of an empire of a specific Tech Level. For example if TL8 is set to 3, this means a TL8 empire expand at an average rate of 3 AU per century.
If the Compact Empires option is checked, there's less chance to see a new colony founded far away from the other ones owned by this empire.
Military Bonus is the Military power bonus given in defence (30 % in attack) to an extra sector race, this represents the reinforcements dispatched by the homeworld to this sector. The standard bonus is set to 17500 because it's what I use for a 30 cubic parsecs sector. If this bonus is too low, the extra sector races could be quickly overhelmed, if it's too high, the native races could have difficulities to match their power. It's with time that you'll find the adequate bonus.
These values have an important effect on Empire results, in a little sector a race with a high Spacial age attribute could quickly colonize it entirely if his TL Range is too high.

RP Log

Checking this box will let CivGen creates a <sector filename>_RP.csv file in which each RP gained by a civilization each century is reported. This file could be read by Excel or Star Office.

Event Log

Checking this box will let CivGen creates a <sector filename>.csv file in whch every event is saved. This file could be read by Excel or Star Office.

CivGen glossary

Allegiance : the race Id that controls the colony.

Captive planets : planets colonized by the alien race that is controlled by an another one.

Captive population : number of population of the alien race that lives in colonies controlled by another race.

GWP : Gross World Product, the economic colony budget in MegaCredits.

Trade bonus: the amount of MegaCredits earned in interstellar commerce with other empires.

Indep. Colonies : number of independant colonies inhabited by the alien race.

Subjects population : number of people not of the same race that lives under the alien race rules.

Subjugated planets : planets conquered by the alien race.

What's wrong with CivGen ?

Well the code is not perfect, and with time it could appear some bugs or oddities.

Some odd things that I found includes :

- an empire with all his colonies conquered (even its homeworld) but a few subjugated colonies. So the only free colonists are not the natives one ! Well this happens for the French during WWII, so why not in space...

- a sector entierly own by one race because it was the first race that reached the stars.

Colonies and empires generated by CivGen could be browsed with the Star Browser v1.6.

Century number

This value is how many centuries should CivGen runs, it must be between 80 and 130. The higher the value, the higher the number of high tech races.

Research Points (RP)

Each century, a race earns between one and ten RP, when the RP amount reaches a certain value, the alien's TL is raised by one and the RP is set down to 0.
RP needed to reach new TL = Base RP x (5 - Progressivness/5)
