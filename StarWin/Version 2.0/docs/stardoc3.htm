Creating Star Systems

On the File menu, click on Generate Sector.

A dialog box will appear with these informations:

Name: name of the file in which datas will be stored
File: click on the edit box to change the file names and the directory where they will be created.
X, Y, Z: coordinates of the first space zone
Density: density of the sector (between 1 and 100 %)
Size: length of the sector in Parsecs, default is 10 (a cube of 10x10x10 parsecs)
Gaia: if checked there is a higher chance of Terran planets
New file: if checked will erase any previous datas contained in the file. If not will merge old systems
with the new created ones.
Tot al disable: to disable the creation of totally alien races.
2D Sector: to create 2D sectors (Z coordinates set to 0, useful if you want to create Traveller like maps).

Create Sector will store datas in files that aren' t humanly readable. You must create a logfile for a system or use Star Browser.

I recommand you to store datafiles in the data directory, and not in the same directory than Star Generator executable file.

Coordinates:

The first space zone created will get the coordinates of (coX,coY,coZ) and the last space zone coordinates will be (coX+9,coY+9,coZ+9) for a sector size of 10. A space zone is a box of 1x1x1 parsec.

1 parsec= 3.26 LY

Density notes:

 

Rift 1 %
Sparse 3 %
Scattered 5 %
Standard 8 %
Dense 15 %

A density of 100% means a star system in each space zone.

Gaia parameter

A user of Star send me a request on how to have more often Terran worlds.
So I decided to include a Gaia parameter who increase the chance of
having Terran planets.Normally there are between 5 and 10 % of chance
to have habitable worlds,when you use the gaia parameter you increase
the chance to between 40 and 50 %.


New Sector

By clicking on New Sector in the File menu you'll generate a blank sector. It's useful if you want to create manually a sector with Star Editor.

Open Sector

By clicking on Open file in the File menu, you'll load a sector.

 

Logfiles

With the first version of Star generator, all datas were stored in logfiles,but with the program Sector generator that could handle thousand of star systems, keeping logfiles for each star systems would have meaned using too much space disk. Now Star generator doesn't keep anymore logfiles, with Browser you could view all the systems created without wasting space disk.


But sometimes you could need to get these datas in an ASCII file (*.log). It's the purpose of System Log and Alien Log (all in the Logs menu). You give the filename and Id number of the star system (or alien race) and Star Generator will create a logfile of the specific system under the log directory (or wherever you want).
With StarGenerator v1.56+ you could convert some of the alien characteristics to a specific rpg system rules. Simply set the Output to the specific game (set to Standard if you doesn't want any conversion).


Sector Report


This utility will give all systems coordinates of a given sector in an ASCII file (*.map) under the log directory. Miscleanous informations about the sector (number and type of stars, planets, alien races...) are given at the end of the file.
With StarGen V1.6+ you could choose what to put in the logfile.
 


Colonies Report

By clicking on the Colonies Report in Logs menu you'll generate a logfile with a list of colonies owned by a specific race (subjugated worlds are included).
 


Merge Sector

With this utility (Utilities menu), you could merge a sector (sector to add) to the end of another one (Main sector).


Alien Generation

By clicking on the Alien Gen button, you will generate a new alien race . A logfile with all its
characteristics will be saved under the file name you gave.


MAP EXPORT

This little tool converts Star Generator datas to *.lst files that could be imported to Chview. Chview is a software that draws 3D map of a local stellar area.

Chview could be downloaded at http://members.nova.org/~sol/chview Chview is a property of Jo Grant and Ben Lin.
 


RPG Conversion

You have the possibility to convert alien race datas to a specific role playing
game system. All the converted characteristics are just after standard alien datas
in the ASCII logfile.

The actually supported rpg are :

Alternity : copyright TSR

Battlelords of the 23rd century : copyright Optimus Design System

Fuzion : copyright Fuzion Labs Group. The fabulous Atomik Alienz plug-in is necessary (http://www.meta-earth.com).

Gurps : copyright Steve Jackson Games

Space Master : copyright Iron Crown Enterprise

Some of the alien abilities are described in the different games rules, for the others
it's up to the GM to specify the ability effect on the rules.

Alternity :

Psionic powers I : the alien Mindwalker has psionic energy points equal to his Will
scored x1.5, and a Diplomat with Mindwalker as his secondary profession, has psionic
energy poins equal to his Will.

Psionic powers II : the alien Mindwalker has psionic energy points equal to his Will
scored x2, and a Diplomat with Mindwalker as his secondary profession, has psionic
energy poins equal to his Will. 

Starwin.ini

This file hold some parameters used by the program.
Here are the standard values:

[Stellar Probabilities]
Pheno=1
Misc=5
A_II=9
M_III=20
F_IV=30
F_V=115
G_V=250
K_V=260
M_V=260
F_VII=50

[Alien Probabilities]
Human=20
Animal=30
Reptile=15
Insect=15
Misc=15
Exotic=5

[Settings]
Height=324
Width=513
Left=196
Top=144

[Browser Settings]
Height=586
Width=786
Left=6
Top=6

Stellar probabilities values represent the probability to get a specific star as the main system star.
The sum of all the values should always be 1000.

Alien probabilities values represent the probability to get a type of species for a standard world.
The sum of all the values should always be 100.

Settings and Browser Settings are the size and position of the applicarions.
